Validation with form

First exapmle is validation with template driven form.
Here we have used classes applied by ngMdel on inpute field. In both template driven or model driven form, we can make use of this classes and show ouyr error messages.
Below are the classe :
1) ng-valid,ng-invalid : whatever validation you have put on your input field, if that is fulfilled by your input field then this ng-valid class will be applied else ng-invalid
2)ng-dirty,ng-pristine : Once a value is entered, then ng-dirty class is added. Now if you remove value from your input field and think that class will be changed again to ng-pristine but it wont happen. your input field is now dirty
3)ng-touched,ng-untouched : once input field is visited, then your field is touched so ng-touched class is applied
